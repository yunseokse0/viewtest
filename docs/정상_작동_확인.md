# ViewBot 정상 작동 확인 방법

솔루션이 제대로 동작하는지 아래 순서대로 확인하세요.

## 1. 서버·웹 연결

1. 터미널에서 `npm run server` 실행 후 **"서버가 http://localhost:3000 에서 실행 중입니다."** 가 보이는지 확인.
2. 브라우저에서 **http://localhost:3000** 접속.
3. **상단에 "모드: 실제 브라우저 (플레이어 로드) · 시청자 수에 반영될 수 있습니다."** 가 보이면 → 서버 연결·백엔드 정상.

※ "요청 전송만" 이나 "서버 연결 실패"면 서버가 꺼져 있거나 다른 포트일 수 있음. `npm run server` 다시 실행.

---

## 2. 봇 시작

1. URL에 **YouTube 라이브 주소** 입력 (예: `https://www.youtube.com/watch?v=...`).
2. 요청 수 **15~20**, 동시 실행 수 **12** 권장.
3. **시작** 클릭.

**정상이면 로그에 이런 순서로 나옵니다.**

| 순서 | 로그 예시 | 의미 |
|------|-----------|------|
| 1 | `시작: https://... (총 20개, 한번에 12명씩)` | 봇 시작 요청 성공 |
| 2 | `총 20개 · 한번에 12명씩 (간격 600ms)` | 동시 실행 설정 적용 |
| 3 | `[인스턴스 1] 브라우저 시작...` | 브라우저 실행됨 |
| 4 | `[인스턴스 1] ... 접속 중...` | 페이지 이동 시도 |
| 5 | `[인스턴스 1] 페이지 로드 완료 - 제목: ...` | 페이지 로드 성공 |
| 6 | `[인스턴스 1] 비디오 재생 중 - 최소 60초 이상 시청` | YouTube 재생 성공 |
| 7 | `[인스턴스 1] 66초 동안 시청 중...` | 시청 시간 진행 |
| 8 | `[인스턴스 1] 세션 종료` | 세션 종료 |
| 9 | `[인스턴스 1] 이 방송에 시청자 1명 반영됨` | 시청자 1명 반영됨 |

위 1~9가 **일부라도 순서대로** 나오면 봇·페이지·재생·반영까지 정상 동작하는 것입니다.

---

## 3. 통계·시청자 수

- **통계**: 목표 요청 수, 진행 중, **완료**, 실패 숫자가 갱신되는지 확인.
- **실행 시간**: 00:00:00 이 1분, 2분… 으로 늘어나는지 확인.
- **YouTube 시청자 통계**: 시작 시 시청자, 현재 시청자, **변화 +N명** 이 나오는지 확인.
- 녹색 메시지 **"이 방송에서 시청자 증가 감지 (+N명)"** 가 나오면 → 시청자 수 반영 정상.

※ 완료 수가 0에서 안 늘거나, "비디오 재생 실패"·"타임아웃"만 많으면 → 동시 실행 수를 8~10으로 낮추거나, PC/네트워크를 여유 있게 한 뒤 다시 시도.

---

## 4. 정상 작동 요약

| 확인 항목 | 정상일 때 |
|----------|-----------|
| 서버 | 터미널에 "서버가 ... 실행 중" |
| 웹 | "모드: 실제 브라우저 (플레이어 로드)" 표시 |
| 시작 | 로그에 "시작: ... (총 N개, 한번에 M명씩)" |
| 브라우저 | "[인스턴스 N] 브라우저 시작...", "페이지 로드 완료" |
| 재생 | "비디오 재생 중 - 최소 60초 이상 시청" |
| 반영 | "이 방송에 시청자 1명 반영됨", 통계 완료 수 증가 |
| 시청자 | YouTube 시청자 통계 "변화 +N명", "시청자 증가 감지" |

위가 대부분 맞으면 **솔루션이 정상 작동**하는 것입니다.
