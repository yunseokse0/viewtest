# 정상 작동하지 않을 때

## 0. "로딩 중"에서 안 넘어갈 때

**증상:** localhost:3000 접속 시 탭이 "로드 중..." 상태로 멈춤, 하얀 화면만 보임.

**원인:** 3000 포트를 잡고 있는 **예전 Node 서버가 멈춰 있어서** 응답을 안 함.

**가장 빠른 해결 – 다른 포트로 실행:**

1. **새 터미널**을 열고 프로젝트 폴더로 이동:
   ```powershell
   cd c:\project\viewtest
   ```
2. **3001 포트**로 서버 실행:
   ```powershell
   npm run server:3001
   ```
   (또는 PowerShell에서: `$env:PORT=3001; node server.js`)
3. 브라우저에서 **http://localhost:3001** 로 접속 (3000이 아님).

이렇게 하면 3000 포트를 정리하지 않아도 바로 사용할 수 있습니다.

---

## 1. 화면이 하얗게만 나올 때 (localhost:3000)

**원인:** 예전에 켜 둔 Node 서버가 3000 포트를 잡은 채 **멈춰 있어서** 새 요청에 응답하지 않는 경우가 많습니다.

**해결:**

1. **작업 관리자** (Ctrl+Shift+Esc) → **세부 정보** → **node.exe** 찾아서 **작업 끝내기**
2. 또는 **PowerShell(관리자)** 에서:
   ```powershell
   netstat -ano | findstr :3000
   ```
   나온 **PID**(맨 오른쪽 숫자) 확인 후:
   ```powershell
   taskkill /PID 그숫자 /F
   ```
3. **프로젝트 폴더**에서 서버 다시 실행:
   ```powershell
   cd c:\project\viewtest
   npm run server
   ```
4. 콘솔에 **"서버가 http://localhost:3000 에서 실행 중입니다."** 가 보이면 브라우저에서 **http://localhost:3000** 또는 **http://127.0.0.1:3000** 접속

**한 번에 재시작:** 프로젝트 폴더에서
```powershell
.\restart-server.ps1
```
(3000 포트 사용 프로세스를 종료한 뒤 서버를 실행합니다.)

---

## 2. "주소 이미 사용 중"(EADDRINUSE)

3000 포트를 다른 프로그램이 쓰고 있습니다. 위 1번처럼 **node.exe**를 종료한 뒤 다시 `npm run server` 하세요.

---

## 3. "public/index.html을 찾을 수 없습니다"

**프로젝트 루트**에서 서버를 실행해야 합니다.

```powershell
cd c:\project\viewtest
npm run server
```

`c:\project\viewtest` 안에 `server.js`와 `public` 폴더가 있어야 합니다.

---

## 4. 서버는 뜨는데 페이지가 안 보일 때

- **http://127.0.0.1:3000** 로도 접속해 보기
- 브라우저 **캐시 비우기** 또는 **시크릿 창**에서 접속
- **개발자 도구(F12)** → **Network** 탭에서 `/` 요청이 **200**으로 오는지 확인

---

## 5. 요약

| 증상           | 조치                                      |
|----------------|-------------------------------------------|
| 하얀 화면      | node.exe 종료 후 `npm run server` 재실행  |
| EADDRINUSE     | 3000 포트 쓰는 프로세스 종료 후 재실행   |
| 파일 없음 오류 | `cd c:\project\viewtest` 후 서버 실행     |
